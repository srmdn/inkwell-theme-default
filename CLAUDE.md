# CLAUDE.md

This file defines reusable collaboration rules for humans and AI tools in this repository.

## Scope

- Applies to all contributors (human and AI-assisted).
- Model/tool agnostic: Claude, ChatGPT, Codex, local models, and future tools.

## Operating Rules

1. Humans own all final decisions and commits.
2. AI output must be reviewed for correctness, security, and licensing.
3. Do not include AI branding/trailers in commit messages.
4. Keep changes small, traceable, and reversible.

## Commit Ownership Standard

- Commit messages must be human-authored.
- Disallowed examples include lines like:
  - `Co-authored-by: Claude ...`
  - `Generated with Claude Code`
  - `Co-authored-by: ChatGPT ...`
  - `Generated by <AI tool/model> ...`

Use `scripts/check-commit-attribution.sh` before push.

## Portable vs Project-Specific

Portable (keep in starter):
- commit attribution rules
- cleanup runbook
- CI checks
- PR/MR templates

Project-specific (customize after copy):
- security contacts
- code owners
- release policy
- escalation and incident contacts

## Project: inkwell-theme-default

Default Astro SSR theme for the Inkwell CMS. Reads post content from the
filesystem via `CONTENT_DIR`. Ships with Inkwell as the reference theme.

## Stack
- Framework: Astro SSR (`@astrojs/node` adapter)
- Language: TypeScript
- Content: reads Markdown files from `CONTENT_DIR` (set in `.env`)
- Styling: plain CSS, no framework

## Environment: LOCAL DEV

## Conventions
- Secrets in `.env` — never committed
- `.env.example` committed with all variable names, no real values
- Build output in `dist/` — gitignored
- No server paths, IPs, or domain names in committed files
- Public repo — keep all content generic and portable

## Do not modify without confirming
- Theme contract interface (frontmatter fields, content dir structure)
  These are defined in the Inkwell core repo `docs/theme-contract.md`
